# ROADMAP — 2026-06-26 21:33 UTC — MLB Betting Data Ingestion Layer

## What was built
An isolated, **data-only** ingestion module in `rambo-backend/`. Free
`statsapi.mlb.com` (roster / schedule / stats) + two paid Apify actors (odds, DK
Pick6 props) → spend-capped landing → `raw_ingest` → per-source normalization →
typed tables → read-only repo → (downstream) EV Brain. No bet-placement capability
anywhere in this module (Sentinel boundary holds by construction — it imports no
betting tools).

```
statsapi (free) ─┐
                 ├─▶ raw_ingest ─▶ normalize ─▶ games / players / *_stats / *_lines ─▶ MlbRepo ─▶ Brains
Apify (paid) ────┘   (sha256,      (per-source   (typed)                              (read-only)
 odds + Pick6         idempotent)   mappers)
```

## Key decisions / deviations from the reference files
- **Free statsapi replaces 3 paid actors.** roster/schedule/stats come direct from
  `statsapi.mlb.com` (no key); only odds + Pick6 stay on paid Apify. Cuts ~60% of spend.
- **`ingestion/sources.py`** added as the one dispatcher both API + CLI call (routes a
  source to the free or paid path).
- Reference bugs fixed against the live runtime:
  - apify-client 3.x: `.call()` uses `run_timeout` (not `timeout_secs`), returns a `Run`
    **object** not a dict; added `max_items` + **`max_total_charge_usd`** (hard spend cap).
  - retry loop restructured so a post-success parse error never **re-runs a paid actor**.
  - `migrate.py` dropped the manual `BEGIN/COMMIT` around `executescript()` (errored
    "no transaction is active" on Python's sqlite3).
  - game-log `game_pk` left NULL (FK to current-slate `games` would fail on historical logs).
- **Field paths corrected to real shapes** (verified from live data / maxItems=1 pulls):
  odds = `bookmaker`/`marketKey`/`outcomeName`/`priceAmerican`/`point`/`eventId`;
  props = `player_name`/`stat_abbr`/`line`/`over_multiplier` (multiplier-only, no price).
- **Odds input corrected** to the actor's schema: `{"leagues": ["mlb"]}` (was defaulting to NFL).
- **Verified pricing:** odds $3.00/1k rows; Pick6 props ~$0.03/1k.

## Schema decisions
- STRICT tables (SQLite 3.50.4 here). Raw lands untouched first; normalization is a
  separate re-runnable pass (watermark `normalized_at`). `mlb_id` is the single
  canonical anchor; cross-feed ids live in `player_aliases`; weak matches → `player_review`.
  Line tables are append-only snapshots deduped to the minute via a STORED `snapshot_key`.
  Stats stored as JSON (season + L/R splits + game logs) so new metrics need no migration.
